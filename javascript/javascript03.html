02. 함수 : 익명 함수

자바스크립트에서 익명 함수(Anonymous Functions)는 이름이 없는 함수를 말합니다.
이러한 함수는 주로 다른 함수의 매개변수로 전달되거나, 함수 내에서 동적으로 생성되는 경우에 사용됩니다. 

{
    const func = function () {
        console.log("02. 함수가 실행되었습니다.");
    }
    func();
}

02. 함수가 실행되었습니다.

03. 함수 : 매개변수 함수

JavaScript에서 함수가 호출될 때 받는 입력값을 말합니다.
함수를 정의할 때, 함수의 동작에 필요한 값을 전달하기 위해 매개변수를 사용합니다.
이 매개변수는 함수의 선언 부분에 정의되며, 함수가 호출될 때 해당 매개변수에 실제 값을 전달합니다. 

{
    // 선언적 함수
    function func(str) {
        console.log(str);
    }

    func("03. 함수가 실행되었습니다.");

    // 익명 함수
    const func1 = function (str) {
        console.log(str);
    }

    func1("03-2. 함수가 실행되었습니다.");
}
결과 확인하기

10. 화살표 함수 : 매개변수 + 리턴값 함수

함수를 간결하게 표현할 수 있어서 코드의 가독성을 높일 수 있습니다.

{
    // 익명 함수
    const func = (str) => {
        return str;
    }

    console.log(func("10. 함수가 실행되었습니다."));

    // 매개변수 하나이면 괄호 생략 가능
    const func2 = str => {
        return str;
    }

    console.log(func2("10-2. 함수가 실행되었습니다."));

    // 리턴 생략
    const func3 = str => str;
    console.log(func3("10-3. 함수가 실행되었습니다."));

    // 선언적 함수 (const 익명 선언 생략) 
    func4 = str => str;
    console.log(func4("10-4. 함수가 실행되었습니다."));
        
}
결과 확인하기
10. 함수가 실행되었습니다.

17. 함수 유형: 객체 생성자 함수

함수를 간결하게 표현할 수 있어서 코드의 가독성을 높일 수 있습니다.
선언적 함수 생략 같은 경우는 코드의 간결성은 뛰어나지만, 가독성이 현저히 떨어져 실제론 잘 사용하지 않습니다. 

{
    // 함수 정의
    function Func(num, str) { // 대문자는 객체 생성자라는 걸 나타내기 위한 것.
        this.num = num; // 자기 자신에 있는 걸 받아오니 this.
        this.str = str;
        this.result = () => {
            console.log(`${num}. ${str}`);
        }
    }

    // 인스턴스 정의
    const info1 = new Func(19, "함수가 실행되었습니다.");
    const info2 = new Func(20, "함수가 실행되었습니다.");

    info1.result();
    info2.result();

}
결과 확인하기
19, 함수가 실행되었습니다.
20, 함수가 실행되었습니다.

{
    // 함수 정의
    function Func(num, str) {
        this.num = num;

    const info2 = new Func(22, "함수가 실행되었습니다.");

    info1.result();
    info2.result();

}

{
    // 함수 정의

    const info2 = new Func(24, "함수가 실행되었습니다.");

    info1.result1();
    info2.result2();
}
결과 확인하기
23, 함수가 실행되었습니다.
24, 함수가 실행되었습니다.
